2. Mechanism

Initially, the system is in a disconnected state where all the hosts are disconnected from the server and each other. When a host wants to connect to the Internet, it has to connect to the Server first. This connection is authenticated with a username and password verification. Once the authentication is successful, the host is connected to the server for a fixed period of time depending on the quota purchased by the customer. Once the session expires, the customer needs to make a fresh payment and the process repeats again.


To establish this captive portal, the following scripts have been developed using python coding:

  1. admin-login.py : This script generates a random 6 digit alphanumeric code for a particular room umber and time quota.
  2. user-login.py : User enters the username and password. Based on the validity of the username and password, the activation status is updated.
  3. hotel_internet.py :The initial states are defined. Connection between mininet and ryu controller is estabished. The server MAC address is defined. The firewall policies are described here. The conditions for connection or disconnection is established.
  4. database: Room number, MAC address, Password, Time period for connection and Activation status are listed in the database.
The following steps take place in the successful execution of this application:
  1. Initially a connection is established between mininet and Ryu controller. A topology is build consisting of a switch, 6 hosts and a server. The firewall policies are entered enabling the switch and making it active. All the hosts are disconnected from the server. Disconnection is established by using post API. All the passwords in the policy are by default set as expired.
